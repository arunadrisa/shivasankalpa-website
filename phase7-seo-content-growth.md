## 1. ON-PAGE SEO PLAN

### Foundation Principles

Your SEO strategy is built on three pillars:
1. **Relevance** — Every page targets specific keywords that match what people actually search for
2. **Trust** — Google rewards sites that demonstrate expertise, authority, and trustworthiness (E-E-A-T)
3. **Growth** — The blog is your long-term SEO engine, pulling in organic traffic through valuable content

### Page-Level SEO Checklist (Apply to Every Page)

| Element | Requirement |
|---------|-------------|
| **Meta Title** | Unique, 50–60 characters, includes primary keyword, ends with brand name |
| **Meta Description** | Unique, 140–160 characters, includes primary keyword, has a call to action |
| **H1 Heading** | Exactly one per page, includes primary keyword |
| **H2 / H3 Headings** | Use for section structure, include secondary keywords naturally |
| **URL Slug** | Short, lowercase, hyphenated, keyword-rich, no stop words |
| **Internal Links** | Every page links to at least 2 other pages on the site |
| **Images** | Compressed, descriptive file names, meaningful alt text |
| **Mobile Friendly** | All content readable and tappable on mobile |
| **Page Speed** | Target < 3 seconds load time (LiteSpeed Cache + ShortPixel handle this) |
| **Yoast SEO** | Green or yellow indicator — never leave red |

### Technical SEO (One-Time Setup)

| Task | How |
|------|-----|
| **XML Sitemap** | Yoast generates automatically at `shivasankalpa.org/sitemap_index.xml` |
| **Submit to Google Search Console** | Create account → add property → verify via DNS (Cloudflare) → submit sitemap |
| **Submit to Bing Webmaster Tools** | Optional but free — import from Google Search Console |
| **Robots.txt** | Yoast manages this. Verify it doesn't block important pages. |
| **Canonical URLs** | Yoast handles automatically. No action needed. |
| **HTTPS** | Already active via Hostinger SSL. Verify padlock on all pages. |
| **Breadcrumbs** | Enable in Yoast → Settings → Breadcrumbs. Astra displays them automatically. |
| **404 Page** | Astra has a default 404. Customize it to include a search bar and link to Home. |
| **Permalink Structure** | Settings → Permalinks → "Post name" (should already be set). Never change this after launch. |

## 4. INTERNAL LINKING STRATEGY

Internal links help Google understand your site structure and help visitors discover more content. Every page should link to at least 2 other pages.

```

├── links to → Gurukulas (why Gurukulas matter section)
└── links to → Blog (if a featured post is added)

ABOUT
├── links to → Events (CTA strip)
├── links to → Donate (CTA strip)
├── links to → Gurukulas (objectives section)
└── links to → Contact (bottom CTA)

EVENTS
├── links to → Maharudra detail page (featured card)
├── links to → Donate (seva links)
└── links to → Contact (volunteer/questions)

├── links to → Donate (seva CTA)
├── links to → About (trust context)
├── links to → Contact (questions)
└── links to → Gallery (after event, link to event photos)

DONATE
├── links to → About (transparency, who we are)
├── links to → Events / Maharudra (seva context)
├── links to → Contact (questions about donations)
└── links to → Gurukulas (where funds go)

GALLERY
├── links to → Events (event galleries link to event pages)
├── links to → Donate (CTA strip)
└── links to → About (trust context)

BLOG POSTS
├── links to → other blog posts in same cluster
├── links to → Donate (CTA at bottom)
├── links to → relevant pages (Events, About, Gurukulas)
└── pillar posts link to → all supporting posts in cluster

GURUKULAS
├── links to → Donate (support this Gurukula)
├── links to → About (trust mission)
└── links to → Contact (know a Gurukula?)

CONTACT
├── links to → Donate
├── links to → Events
└── links to → About
```

### Blog Internal Linking Rules

1. **Every blog post** links to at least 1 other blog post and 1 main page (Donate, Events, or About)
2. **Pillar posts** link to all supporting posts in their cluster
3. **Supporting posts** link back to their pillar post
4. **Use descriptive anchor text** — not "click here" but "learn more about the Guru–Shishya Paramparā"
5. **When publishing a new post**, go back and add a link to it from 1–2 existing related posts

## 6. IMAGE ALT TEXT GUIDELINES

Alt text serves two purposes: accessibility (screen readers) and SEO (Google image search).

### Rules

| Rule | Example |
|------|---------|
| **Be descriptive** | "Young students performing Vedic chanting during morning prayers at a Gurukula" |
| **Include context** | "Rudra Homa fire ceremony during Maharudra Purascharana 2026" |
| **Include keywords naturally** | "Vedic Gurukula students studying Vedangas with their Adhyapaka" |
| **Keep it under 125 characters** | Long enough to describe, short enough to be useful |
| **Don't start with "Image of" or "Photo of"** | Screen readers already announce it as an image |
| **Leave empty for decorative images** | Background textures, dividers, purely decorative elements |

### Alt Text Templates by Page

| Page | Template |
|------|----------|
| Home hero | "[Scene description] at a Vedic Gurukula" |
| Trustee photos | "Sri [Name], Trustee of Sri Shivasankalpa Vṛnda" |
| Event images | "[Activity] during [Event Name] at [Location]" |
| Gallery images | "[Descriptive scene] — [category context]" |
| Blog featured images | "[Topic-relevant description] — Sri Shivasankalpa Vṛnda" |

### Common Mistakes to Avoid

| Mistake | Why It's Bad |
|---------|-------------|
| `alt=""` on meaningful images | Invisible to Google and screen readers |
| `alt="image1"` or `alt="IMG_3847"` | Zero SEO value, meaningless |
| `alt="Vedic Vedic Vedas Vedangas Gurukula Dharma"` | Keyword stuffing — Google penalizes this |
| Same alt text on every image | Looks spammy to search engines |

| Include year for dated content | `/events/ganesha-homa-august-2026` | `/events/ganesha-homa` |
| Never change a published slug | — | Breaks SEO and all links |

## 8. CATEGORY & TAG STRATEGY

### Blog Categories (Fixed, Broad)

| Category | Slug | Purpose |
|----------|------|---------|
| Trust Updates | `trust-updates` | News, announcements, reports, partnership updates |
| Vedic Education | `vedic-education` | Explainers about Gurukula system, Vedas, Vedangas, Shastras |
| Dharmic Thought | `dharmic-thought` | Reflections on Vedic Dharma, philosophy, spiritual concepts |
| Events | `events` | Event announcements, recaps, behind-the-scenes |

**Rules:**
- Assign exactly **one category** per post
- Categories should be stable — don't create new ones for every post
- Only add a new category when you have 5+ posts that would belong to it

### Blog Tags (Flexible, Specific)

| Tag | Use When Post Mentions |
|-----|----------------------|
| Maharudra | Maharudra Purascharana specifically |
| Rudra | Rudra, Rudradhyaya, Rudrabhisheka |
| Gurukula | Any specific Gurukula or Gurukula system |
| Vedas | Vedic texts, Vedadhyayana |
| Vedangas | Six Vedangas, specific Vedanga topics |
| Sringeri | Sringeri Peetham, Jagadgurus |
| Homa | Any Homa or fire ritual |
| Adhyapaka | Teachers, scholars |
| Go-samrakshana | Goshala, cow protection |
| Bangalore | Events or activities in Bangalore |

**Rules:**
- Use **2–4 tags** per post, never more than 5
- Reuse existing tags — check the tag list before creating a new one
- Tags should be words/phrases people might search for
- Don't duplicate categories as tags (no "Vedic Education" tag if that's already a category)

## 9. LAUNCH SEO CHECKLIST

Complete this checklist before or immediately after going live.

### Pre-Launch (Day 7 of build)

- [ ] **Permalink structure** set to "Post name" (Settings → Permalinks)
- [ ] **Yoast SEO** configured — organization name, logo, social profiles
- [ ] **XML Sitemap** active — verify at `shivasankalpa.org/sitemap_index.xml`
- [ ] **Meta titles and descriptions** set for ALL pages (from Phase 4)
- [ ] **Focus keyphrase** set for each page in Yoast
- [ ] **H1 headings** — exactly one per page, contains primary keyword
- [ ] **All images** have descriptive alt text
- [ ] **All images** compressed (ShortPixel active)
- [ ] **Internal links** — every page links to at least 2 other pages
- [ ] **Breadcrumbs** enabled in Yoast
- [ ] **SSL active** — padlock visible on all pages
- [ ] **Mobile responsive** — test all pages on phone
- [ ] **Page speed** — test at pagespeed.web.dev, aim for 70+ on mobile
- [ ] **404 page** — visit a non-existent URL, confirm it shows a helpful error page
- [ ] **Robots.txt** — verify at `shivasankalpa.org/robots.txt`, ensure no pages are accidentally blocked
- [ ] **Favicon** uploaded (Astra Customizer → Site Identity → Site Icon)

### Post-Launch (Week 1 after going live)

- [ ] **Google Search Console** — create property, verify domain via Cloudflare DNS, submit sitemap
- [ ] **Google Analytics** — create GA4 property, add tracking code via WPCode plugin
- [ ] **Bing Webmaster Tools** — import from Search Console (optional but free)
- [ ] **Test site search** — Google `site:shivasankalpa.org` to check indexing (may take a few days)
- [ ] **Social sharing test** — share the homepage URL on WhatsApp/Twitter, verify the preview card looks correct (image, title, description)
- [ ] **Open Graph tags** — Yoast handles this automatically, but verify the preview image is correct per page

### Ongoing (Monthly)

- [ ] **Check Search Console** for crawl errors, indexing issues, search performance
- [ ] **Review top queries** — what are people searching to find you?
- [ ] **Publish 2+ blog posts** per month with proper SEO
- [ ] **Add internal links** from new posts to existing content and vice versa
- [ ] **Update meta descriptions** if click-through rates are low for any page
- [ ] **Monitor page speed** if new images/content are added heavily

## SEO GROWTH TIMELINE — REALISTIC EXPECTATIONS

| Period | What to Expect |
|--------|---------------|
| **Month 1** | Google indexes your pages. Brand searches ("Shivasankalpa") start showing results. Very little organic traffic from non-brand queries. |
| **Month 2–3** | Event-related searches may surface ("Maharudra Purascharana 2026"). Blog posts begin getting indexed. |
| **Month 3–6** | Long-tail queries start bringing traffic ("what is Rudrabhisheka", "Vedic Gurukula system"). Blog clusters gain traction. |
| **Month 6–12** | Consistent blog publishing builds authority. Medium-competition keywords start ranking. Organic traffic grows steadily. |
| **Year 2+** | Domain authority builds. Higher-competition keywords like "Vedic education" or "support Gurukula" become realistic targets. |

**Key insight:** SEO is a long game. The first 3 months are about indexing and foundation. Real organic traffic growth starts around month 4–6 with consistent content publishing. Paid or social promotion accelerates early visibility while SEO builds.
